Profil privé — que voit un visiteur ?

Quand un utilisateur active "Profil privé" dans ses settings

A

Page bloquée

La page /u/username retourne une erreur "This profile is private" — nom + avatar visible, rien d'autre. Les composants ne sont pas accessibles depuis le profil, mais restent accessibles via leur lien direct /c/slug s'ils sont publics.

B

Profil masqué + composants publics toujours accessibles

Le profil /u/username est introuvable (404 ou "Private profile"). Mais les composants publics restent accessibles via /c/slug et apparaissent dans le browse. Le créateur est anonymisé dans le browse ("Private user").

C

Profil + composants tous masqués

Profil privé = tout est caché. La page /u/username retourne 404, et les composants publics de cet utilisateur disparaissent du browse et de leurs liens directs. Uniquement visible par le propriétaire connecté.
